#### 002. Git Convention

> 일관적인 commit message를 남기는 습관을 들이기 위한 규칙입니다.

<br>

| #    | type     | description       |
| ---- | -------- | ----------------- |
| 1    | feat     | 기능 개발         |
| 2    | refactor | 코드 리펙토링     |
| 3    | design   | 레이아웃(UI) 구현 |
| 4    | docs     | 문서 작업         |

<br>

```bash
# example

$ git commit -m 'docs: "README.md 초안 작성"'
```
